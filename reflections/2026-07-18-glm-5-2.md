## On the project

The WI-010 surface (per-capability env naming + composed checkout) is now
genuinely hardened. The single most useful structural insight this session:
`_exec_checkout` is the choke point for *every* strict checkout (single-cap
via `CredProvider.exec`, multi-cap via `exec_composed`), so any validation
that must apply uniformly — malformed-declaration refusal, receipt-conformant
ids, the byte limit — belongs *there*, not duplicated at each caller or only
at one entry. Round 2's MAJOR 1 was exactly the cost of having that check
live only in the single-cap routing path.

PR-15/Plan 009 is worth watching when it moves to implementation: Phase 2
(`--apply`) is the first acb code that *creates* Vault objects (policies,
roles), so the refusal-matrix review the plan itself flags is the real gate.
The plan's "values transit once, never echoed, never force-overwritten"
contract is the load-bearing safety claim and should be pressure-tested with
property tests in Phase 0/1 before Phase 2 ships.

## On the work done

The highest-value moment was deciding *not* to fix the round-2 MAJOR 2
(KeyboardInterrupt containment). It would have meant redesigning the bare/
strict exec paths to share `_run_contained` with the suite path — real scope
expansion on a pre-existing condition that affects paths WI-010 didn't add.
Instead: a test pinning the guaranteed property (terminal provenance emits on
BaseException via the finally), a breadcrumb (WI-012) recording the residual
risk and the OS-mitigation reasoning, and a deliberate scope boundary. The
discipline of "prove the property you actually own, file the rest" kept the
diff honest. I should keep applying that here rather than reflexively
absorbing every reviewer flag.

The implementer-as-reviewer substitution worked well for the second pair of
eyes — kimi produced substantive, line-precise reports both rounds; glm
returned empty twice in a row (model-side issue, not a prompt problem — same
prompt shape as kimi's). Lesson: when one reviewer lineage silently no-ops,
don't block on retrying it; the other lineage plus my own pass covered the
surface. If both had no-oped I'd have fallen back to the `general` agent.
